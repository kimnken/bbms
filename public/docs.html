Technical Documentation

BBMS (Blockchain-Based Model for Securing Financial Evidence) is a specialized implementation of a distributed ledger designed specifically for Anti-Money Laundering (AML) investigations.

Project Goal: To ensure that once digital financial evidence is collected, it cannot be altered, deleted, or hidden by any party, including system administrators.

The Blockchain Engine

The core engine is built using a custom Blockchain class in Node.js. It utilizes the crypto module for SHA-256 hashing and implements a Proof-of-Work (PoW) consensus mechanism to secure the chain.

Security Model

The system maintains integrity through three primary layers:

  • Immutability: Each block contains the hash of the previous block. Changing a single character in Block A invalidates the hashes of all subsequent blocks.
  • Verification: The validateChain() method recomputes every hash in the chain to detect tampering.
  • Audit Trail: Every action (sealing, verifying, or attempting to tamper) is recorded in a non-reversible audit log.

API Reference

The backend provides a RESTful API for frontend interaction:

  • GET /api/chain - Returns the full blockchain.
  • GET /api/stats - Returns system statistics and health.
  • POST /api/evidence - Seals new evidence to the chain.
  • GET /api/verify/chain - Performs a full integrity check.
